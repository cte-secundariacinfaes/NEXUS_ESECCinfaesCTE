🔌 Terminal RFID ESP32 — Nexus ESECCinfãesCTE

Terminal de presenças por cartão RFID para o sistema Nexus. O aluno passa o cartão ao entrar na sala ou no estágio — o ESP32 lê o UID, comunica com o Google Sheets Mestre via WiFi, e regista a presença automaticamente.

💡
O terminal é complementar ao sistema de telemóvel — ambos podem coexistir. Se o terminal falhar, os alunos usam o telemóvel como fallback.

🏗️ Arquitetura

Diagrama ┌─────────────────┐ WiFi HTTP POST ┌──────────────────────┐ │ ESP32 Terminal │ ──────────────────────→ │ Code_Mestre.gs │ │ + RC522 RFID │ ←────────────────────── │ (Google Apps Script) │ │ + OLED SSH1106 │ JSON response └──────────┬───────────┘ │ + Buzzer + LEDs │ │ └─────────────────┘ ┌────────▼───────────┐ │ Code.gs (Turma) │ │ → Presenças Sheets │ └────────────────────┘ Fluxo: 1. Aluno passa cartão → ESP32 lê UID (ex: A3B2C1D4) 2. ESP32 → POST {action:"rfidPresenca", uid:"A3B2C1D4", token:"..."} 3. Mestre resolve UID → nome, turma, sheetsUrl 4. Mestre → POST para Code.gs da turma → regista presença 5. Mestre → responde ao ESP32: {ok:true, nome:"David Cardoso"} 6. ESP32 mostra no OLED: "✓ David Cardoso" + bip verde

🔀 Modos de Operação

ModoDescriçãoQuando usar
EntradaRegista entrada do alunoInício da aula / estágio
SaídaRegista saída do alunoFim da aula / estágio
RegistoMostra UID do cartão (para associar a aluno)Configuração inicial

O modo é alterado com o botão físico no terminal (GPIO 0 / BOOT). Cada pressão alterna: Entrada → Saída → Registo → Entrada → ...

⚠️
RC522 funciona a 3.3V! Nunca ligar o VCC do RC522 a 5V — danifica o módulo. O ESP32 DevKit fornece 3.3V no pino 3V3.

🔧 Montagem Passo a Passo

  1. Coloca o ESP32 DevKit na breadboard, centrado, com os pinos a ocupar ambos os lados
  2. Liga o RC522 com 7 jumpers M-F conforme a tabela acima. O fio mais crítico é o 3V3 — confirma que não está nos 5V
  3. Liga o OLED SSH1106 com 4 jumpers: VDD→3V3, GND→GND, SCK→GPIO22, SDA→GPIO21
  4. Liga o buzzer: pino longo (+) ao GPIO 25, pino curto (−) ao GND
  5. Liga o LED verde: ânodo → resistência 220Ω → GPIO 26, cátodo → GND
  6. Liga o LED vermelho: ânodo → resistência 220Ω → GPIO 33, cátodo → GND
  7. O botão de modo usa o botão BOOT do ESP32 (GPIO 0) — já está na placa, não precisa de fio extra
  8. Liga o cabo USB ao computador para programar
✅
Antes de programar, verifica: 1) RC522 ligado a 3.3V (não 5V), 2) SDA do OLED no GPIO 21 (não confundir com SDA do RC522 que é GPIO 5), 3) LEDs com resistência (sem resistência queimam).

⚙️ Configuração do Arduino IDE

  1. Instala o Arduino IDE 2.x
  2. Adiciona o ESP32 ao Arduino IDE:
    File → Preferences → Additional Board URLs:
    https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  3. Tools → Board → Boards Manager → procura esp32 → instala esp32 by Espressif
  4. Seleciona a board: Tools → Board → ESP32 Dev Module
  5. Instala as bibliotecas: Sketch → Include Library → Manage Libraries
    BibliotecaAutorPara quê
    MFRC522GithubCommunityLeitor RFID RC522
    U8g2olikrausOLED SSH1106 (I2C)
    ArduinoJsonBenoit BlanchonParsing JSON
    WiFiClientSecure(built-in ESP32)HTTPS para Google
  6. Seleciona a porta: Tools → Port → COM3 (ou /dev/ttyUSB0 em Linux)

💾 Firmware Completo

Copia o código abaixo, ajusta as 3 constantes no topo (WiFi + URL + Token), e faz upload.

Arduino (C++) // ═══════════════════════════════════════════════════════════════ // Nexus ESECCinfãesCTE — Terminal RFID ESP32 (v1.0) // ═══════════════════════════════════════════════════════════════ #include <WiFi.h> #include <HTTPClient.h> #include <WiFiClientSecure.h> #include <SPI.h> #include <MFRC522.h> #include <U8g2lib.h> #include <ArduinoJson.h> #include <Wire.h> // ── CONFIGURAÇÃO (ALTERAR ESTES 3 VALORES) ────────────────────── const char* WIFI_SSID = "NOME_DA_REDE"; const char* WIFI_PASS = "PASSWORD_WIFI"; const char* MESTRE_URL = "https://script.google.com/macros/s/AAAA.../exec"; const char* API_TOKEN = "nexus_eac_2026_token"; const char* TERMINAL_NAME = "Lab Eletrónica"; // nome deste terminal // ── PINOS ──────────────────────────────────────────────────────── #define RC522_SS 5 #define RC522_RST 27 #define BUZZER 25 #define LED_GREEN 26 #define LED_RED 33 #define BTN_MODE 0 // BOOT button // ── OBJETOS ────────────────────────────────────────────────────── MFRC522 rfid(RC522_SS, RC522_RST); U8G2_SH1106_128X64_NONAME_F_HW_I2C oled(U8G2_R0, U8X8_PIN_NONE); // ── ESTADO ─────────────────────────────────────────────────────── enum Mode { MODE_ENTRADA, MODE_SAIDA, MODE_REGISTO }; Mode currentMode = MODE_ENTRADA; const char* modeNames[] = {"ENTRADA", "SAIDA", "REGISTO"}; const char* modeIcons[] = {">>", "<<", "ID"}; unsigned long lastCardTime = 0; const unsigned long CARD_COOLDOWN = 3000; // 3s entre leituras // ── SETUP ──────────────────────────────────────────────────────── void setup() { Serial.begin(115200); // Pinos pinMode(BUZZER, OUTPUT); pinMode(LED_GREEN, OUTPUT); pinMode(LED_RED, OUTPUT); pinMode(BTN_MODE, INPUT_PULLUP); // LEDs off digitalWrite(LED_GREEN, LOW); digitalWrite(LED_RED, LOW); // OLED oled.begin(); oled.setFont(u8g2_font_helvB08_tr); showScreen("Nexus RFID", "A iniciar...", ""); // SPI + RFID SPI.begin(); rfid.PCD_Init(); delay(100); // WiFi showScreen("WiFi", "A ligar...", WIFI_SSID); WiFi.begin(WIFI_SSID, WIFI_PASS); int attempts = 0; while (WiFi.status() != WL_CONNECTED && attempts < 30) { delay(500); attempts++; Serial.print("."); } if (WiFi.status() == WL_CONNECTED) { beepOk(); showScreen("WiFi OK", WiFi.localIP().toString().c_str(), "Pronto!"); Serial.println("\nWiFi connected: " + WiFi.localIP().toString()); } else { beepError(); showScreen("WiFi FALHOU", "Sem ligacao", "Verificar rede"); Serial.println("\nWiFi FAILED"); } delay(1500); showIdle(); } // ── LOOP ───────────────────────────────────────────────────────── void loop() { // Verificar botão de modo if (digitalRead(BTN_MODE) == LOW) { delay(50); // debounce if (digitalRead(BTN_MODE) == LOW) { currentMode = (Mode)((currentMode + 1) % 3); beepMode(); showIdle(); while (digitalRead(BTN_MODE) == LOW) delay(10); // esperar soltar delay(200); } } // Verificar WiFi if (WiFi.status() != WL_CONNECTED) { WiFi.reconnect(); delay(1000); return; } // Verificar cartão RFID if (!rfid.PICC_IsNewCardPresent() || !rfid.PICC_ReadCardSerial()) { return; } // Cooldown (evitar leituras duplicadas) if (millis() - lastCardTime < CARD_COOLDOWN) { rfid.PICC_HaltA(); rfid.PCD_StopCrypto1(); return; } lastCardTime = millis(); // Ler UID String uid = ""; for (byte i = 0; i < rfid.uid.size; i++) { if (rfid.uid.uidByte[i] < 0x10) uid += "0"; uid += String(rfid.uid.uidByte[i], HEX); } uid.toUpperCase(); Serial.println("Card UID: " + uid); rfid.PICC_HaltA(); rfid.PCD_StopCrypto1(); // Processar if (currentMode == MODE_REGISTO) { handleRegisto(uid); } else { handlePresenca(uid); } } // ── PRESENÇA ───────────────────────────────────────────────────── void handlePresenca(String uid) { showScreen("A verificar...", uid.c_str(), ""); digitalWrite(LED_GREEN, HIGH); // Build JSON String tipo = (currentMode == MODE_ENTRADA) ? "entrada" : "saida"; String json = "{\"action\":\"rfidPresenca\"," "\"uid\":\"" + uid + "\"," "\"tipo\":\"" + tipo + "\"," "\"terminal\":\"" + String(TERMINAL_NAME) + "\"," "\"token\":\"" + String(API_TOKEN) + "\"}"; // POST to Mestre String response = postToMestre(json); if (response.length() == 0) { beepError(); showResult(false, "Sem resposta", "Verificar WiFi"); return; } // Parse response StaticJsonDocument<512> doc; DeserializationError err = deserializeJson(doc, response); if (err) { beepError(); showResult(false, "Erro JSON", err.c_str()); return; } bool ok = doc["ok"] | false; const char* nome = doc["nome"] | "?"; const char* msg = doc["msg"] | "Erro"; if (ok) { beepOk(); showResult(true, nome, tipo == "entrada" ? "ENTRADA OK" : "SAIDA OK"); } else { beepError(); showResult(false, msg, uid.c_str()); } } // ── REGISTO (mostra UID para copiar) ───────────────────────────── void handleRegisto(String uid) { beepMode(); showScreen("UID do Cartao:", uid.c_str(), "Copiar para Sheets"); Serial.println("REGISTO UID: " + uid); delay(5000); showIdle(); } // ── HTTP POST ──────────────────────────────────────────────────── String postToMestre(String jsonPayload) { WiFiClientSecure client; client.setInsecure(); // Google Apps Script usa HTTPS com redirect HTTPClient http; http.setFollowRedirects(HTTPC_STRICT_FOLLOW_REDIRECTS); http.setTimeout(15000); if (!http.begin(client, MESTRE_URL)) { Serial.println("HTTP begin failed"); return ""; } http.addHeader("Content-Type", "application/json"); int code = http.POST(jsonPayload); String response = ""; if (code == 200 || code == 302) { response = http.getString(); } else { Serial.println("HTTP error: " + String(code)); } http.end(); return response; } // ── OLED DISPLAY ───────────────────────────────────────────────── void showScreen(const char* line1, const char* line2, const char* line3) { oled.clearBuffer(); oled.setFont(u8g2_font_helvB10_tr); oled.drawStr(0, 14, line1); oled.setFont(u8g2_font_helvR08_tr); oled.drawStr(0, 32, line2); oled.drawStr(0, 48, line3); oled.sendBuffer(); } void showIdle() { oled.clearBuffer(); // Header oled.setFont(u8g2_font_helvB10_tr); oled.drawStr(0, 14, "Nexus RFID"); // Mode indicator oled.setFont(u8g2_font_helvB08_tr); oled.drawStr(90, 14, modeIcons[currentMode]); // Divider oled.drawHLine(0, 18, 128); // Mode name oled.setFont(u8g2_font_helvR08_tr); oled.drawStr(0, 32, "Modo:"); oled.setFont(u8g2_font_helvB08_tr); oled.drawStr(35, 32, modeNames[currentMode]); // Terminal name oled.setFont(u8g2_font_helvR08_tr); oled.drawStr(0, 46, TERMINAL_NAME); // Instruction oled.drawStr(0, 60, "Passa o cartao..."); // WiFi indicator if (WiFi.status() == WL_CONNECTED) { oled.drawStr(100, 60, "WiFi"); } else { oled.drawStr(100, 60, "----"); } oled.sendBuffer(); } void showResult(bool success, const char* line1, const char* line2) { oled.clearBuffer(); oled.setFont(u8g2_font_helvB14_tr); if (success) { oled.drawStr(0, 18, "\x2713"); // checkmark } else { oled.drawStr(0, 18, "X"); } oled.setFont(u8g2_font_helvB10_tr); // Truncate line1 to fit OLED char buf[22]; strncpy(buf, line1, 21); buf[21] = '\0'; oled.drawStr(0, 38, buf); oled.setFont(u8g2_font_helvR08_tr); oled.drawStr(0, 54, line2); oled.sendBuffer(); // LEDs if (success) { digitalWrite(LED_GREEN, HIGH); digitalWrite(LED_RED, LOW); } else { digitalWrite(LED_GREEN, LOW); digitalWrite(LED_RED, HIGH); } delay(3000); // Reset digitalWrite(LED_GREEN, LOW); digitalWrite(LED_RED, LOW); showIdle(); } // ── BUZZER ─────────────────────────────────────────────────────── void beepOk() { tone(BUZZER, 1000, 100); delay(120); tone(BUZZER, 1500, 100); delay(120); noTone(BUZZER); } void beepError() { tone(BUZZER, 400, 200); delay(250); tone(BUZZER, 300, 300); delay(350); noTone(BUZZER); } void beepMode() { tone(BUZZER, 800, 80); delay(100); noTone(BUZZER); }

🔧 Configuração WiFi e URL

No topo do firmware, altera as 3 constantes:

const char* WIFI_SSID = "NomeDaRedeWiFi"; // rede WiFi da escola const char* WIFI_PASS = "PasswordDoWiFi"; // password da rede const char* MESTRE_URL = "https://script.google.com/macros/s/AKfyc.../exec"; // URL do Mestre const char* API_TOKEN = "nexus_eac_2026_token"; // mesmo token do sistema const char* TERMINAL_NAME = "Lab Eletrónica"; // nome identificador
⚠️
O API_TOKEN tem de ser exatamente igual ao que está no Code_Mestre.gs, index.html, Professor.html e Aluno.html. Se mudaste o token no sistema, muda aqui também.

💳 Registar Cartões dos Alunos

  1. Liga o terminal ao USB e abre o Serial Monitor (115200 baud)
  2. Pressiona o botão BOOT 2 vezes para entrar no modo REGISTO (o ecrã mostra "REGISTO")
  3. Pede a cada aluno para passar o cartão
  4. O OLED mostra o UID (ex: A3B2C1D4) e o Serial Monitor imprime REGISTO UID: A3B2C1D4
  5. Copia o UID para o Sheets Mestre → coluna K (UID RFID) → na linha do aluno correspondente
  6. Repete para cada aluno
  7. Pressiona BOOT para voltar ao modo ENTRADA
💡
Os cartões de estudante portugueses (cartão de cidadão jovem, cartões escolares) usam MIFARE Classic 1K (13.56 MHz) — compatíveis com o RC522. Cartões de transportes (Andante, Viva Viagem) não funcionam — usam protocolo diferente.

🏫 Setup por Estação — Passo a Passo

Estação 1: Sala de Aula (Lab Eletrónica)

  1. Monta o hardware conforme a secção Montagem
  2. No firmware, configura:
    const char* WIFI_SSID = "EscolaWiFi"; const char* WIFI_PASS = "pass_escola"; const char* MESTRE_URL = "https://script.google.com/macros/s/AKfyc.../exec"; const char* TERMINAL_NAME = "Lab Eletrónica";
  3. Faz upload do firmware (Arduino IDE → botão Upload)
  4. Verifica: OLED mostra "Nexus RFID", "Modo: ENTRADA", "WiFi"
  5. Regista os cartões dos alunos (secção Registar Cartões)
  6. Fixa o terminal junto à porta da sala (velcro, suporte impresso, ou caixa)
  7. Liga à corrente USB

Estação 2: Local de Estágio (Empresa X)

  1. Monta um segundo terminal (mesmos componentes)
  2. Configura WiFi da empresa (pedir ao responsável):
    const char* WIFI_SSID = "EmpresaX_WiFi"; const char* WIFI_PASS = "pass_empresa"; const char* TERMINAL_NAME = "Estágio - Empresa X";
  3. O MESTRE_URL e API_TOKEN são os mesmos — é o mesmo sistema
  4. Faz upload e testa
  5. Se a empresa não tiver WiFi, usa o hotspot do telemóvel do professor ou uma bateria + hotspot dedicado

Estação 3: Segunda Sala (opcional)

Mesmo processo — só muda o TERMINAL_NAME e eventualmente o WiFi. Todos os terminais apontam para o mesmo Sheets Mestre.

✅
Não há limite de terminais. Cada um é independente — se um falhar, os outros continuam. E o sistema de telemóvel (Aluno.html) funciona sempre como fallback.

📡 API do Mestre (Referência)

rfidPresenca

POST // Request { "action": "rfidPresenca", "uid": "A3B2C1D4", "tipo": "entrada", // "entrada" ou "saida" "terminal": "Lab Eletrónica", "token": "nexus_eac_2026_token" } // Response (sucesso) { "ok": true, "msg": "Presença registada", "nome": "David Cardoso", "turma": "3.ºB EAC", "id": "12045" } // Response (erro) { "ok": false, "msg": "Cartão não registado." }

rfidLookup (apenas consulta, sem registar)

📦 Caixa e Montagem Final

Opção A — Caixa plástica simples

  • Caixa ABS 100×60×25mm
  • Recorte retangular no topo para o OLED (35×18mm)
  • Furo lateral para o cabo USB
  • RC522 colado na face interior da tampa (lê através do plástico fino)
  • LEDs e buzzer visíveis/audíveis do exterior

Opção B — Impressão 3D

  • Caixa com slot para OLED, abertura para RC522, furos para LEDs
  • Suporte de parede integrado (velcro ou parafusos)
  • Ficheiro STL personalizável (futuro)

Opção C — Perfboard permanente

  • Solda os componentes numa placa perfurada 5×7cm usando o Perfboard Designer
  • Headers fêmea para o ESP32 (removível para reprogramar)
  • Conectores JST para OLED e RC522 (fácil manutenção)
🛠️
Para uso escolar, a Opção A (caixa plástica) é a mais prática e rápida. A Opção C (perfboard) pode ser um projeto prático para os alunos do curso EAC.
