1 Introdução e Conceito

O Esquemas de Ligação é um software de diagramas de cablagem que permite desenhar as ligações elétricas entre microcontroladores, sensores, atuadores e outros componentes. Funciona 100% no browser, sem instalação, e guarda projetos localmente.

Para que serve?

Este software responde à pergunta "que pino liga onde?". Antes de montar um circuito numa breadboard ou perfboard, o aluno desenha aqui o esquema de ligações — qual pino do Arduino liga a qual pino do sensor, que fios usar, que protocolo de comunicação (I2C, SPI, UART).

Diferença entre esquema de ligações e layout perfboard

2 Interface do Programa

A interface está dividida em três zonas principais:

Barra superior (header)

Contém o nome do projeto (editável), botões de guardar/abrir/exportar. É idêntica ao Perfboard Designer para manter consistência.

Sidebar esquerda

Organizada em secções verticais:

SecçãoConteúdo
FerramentasSelecionar (mover/editar), Ligar pinos (criar fios), Borracha (apagar)
Adicionar componenteBotão para abrir a biblioteca. Botões para criar MCU ou componente custom.
Cabo / FioAparece ao selecionar "Ligar pinos". Nome do cabo, cor (14 presets).
Vista dos fiosAlternar entre modo Linhas e modo Tags.
EdiçãoDesfazer, Refazer, Apagar seleção, Limpar tudo.
Imprimir / ExportarImprimir esquema, exportar HTML, SVG e PNG.
SelecionadoInformação do componente selecionado (nome, categoria, pinos, ligações, lista de fios com AWG). Botão para editar pinos.
AvisosPainel de alertas de validação elétrica e recomendações por componente (aparece automaticamente).

Canvas central (área de trabalho)

A área onde se colocam e ligam os componentes. Fundo escuro com grelha de pontos. Suporta pan (arrastar o fundo) e zoom (scroll do rato). No canto superior direito há botões de zoom (+/−) e reset.

Canvas central

Ao passar o cursor sobre um pino, aparece um destaque circular com tooltip mostrando o nome do pino — confirma visualmente que está a apontar corretamente antes de clicar. Suporta zoom com scroll do rato, pinch zoom em tablets, botões +/− e botão "Ajustar" (zoom-to-fit que centra todos os componentes).

Barra de estado (rodapé)

Mostra a ferramenta ativa, o modo de visualização (Linhas/Tags), número de componentes e ligações, e o nome do projeto.

4 Criar Componentes Custom

Se o componente que precisa não existe na biblioteca, pode criar um personalizado — tanto MCU/placas como sensores/atuadores.

Criar um MCU / Placa

Clicar em "✚ Criar MCU / Placa" na sidebar
Preencher nome (ex: "XIAO nRF52840"), categoria e cor
Na secção Pinos Esquerdos, definir cada pino: nome (ex: "D0") e tipo (Digital, I2C, etc.)
Na secção Pinos Direitos, definir os pinos do lado oposto
Usar "+ Pino" para adicionar mais pinos e ✕ para remover
Clicar "💾 Guardar" — o componente fica disponível na biblioteca

Criar um componente (sensor/atuador)

O processo é idêntico, mas usa-se o botão "✚ Criar componente". Componentes simples (sensores I2C, por exemplo) normalmente têm pinos só num lado (esquerdo), enquanto drivers (A4988, L298N) podem ter pinos dos dois lados.

ℹ️ Onde ficam guardados?
Os componentes personalizados ficam no localStorage do browser — persistem entre sessões mas são locais ao computador. Para partilhar, use a função "📥 Ficheiro" que inclui os componentes custom no JSON exportado.

Dicas para pinagem

✅ Boa prática

Colocar VCC e GND no topo, seguidos pelos pinos de comunicação (I2C, SPI), e depois os pinos digitais/analógicos. Usar nomes iguais aos do datasheet.

⚠️ Evitar

Nomes genéricos como "Pino1", "Pino2". Usar os nomes reais: "VCC", "GND", "SDA", "TRIG", "ECHO", etc.

5 Ligar Pinos (Cabos)

Criar ligações entre componentes é a função central do programa.

Passos para ligar

Selecionar a ferramenta ⚡ Ligar pinos na sidebar
Definir o nome do cabo (ex: "SDA", "VCC", "SENSOR_TRIG") no campo de texto
Escolher a cor — clicar num preset (VCC, GND, SDA, SCL...) ou escolher manualmente. O preset preenche automaticamente o nome.
Clicar no primeiro pino (aparece um círculo pulsante a confirmar)
Clicar no segundo pino (de outro componente) — a ligação é criada
Repetir para todas as ligações do circuito

Cores pré-definidas

VCC VermelhoAlimentação positiva (5V, 3.3V)
GND AzulMassa (0V)
SDA VerdeI2C dados
SCL AmareloI2C relógio
TX / RX Roxo / CyanUART transmissão / receção

SIG1–SIG4DiversasSinais genéricos (TRIG, ECHO, IN1...)
💡 Dica: nomes personalizados
Pode escrever qualquer nome no campo "Nome do cabo". Por exemplo, "MOTOR_DIR", "BUZZER", "LED_R". Este nome aparece na etiqueta do fio e na tabela de ligações ao imprimir. Se não escrever nada, o sistema gera uma tag automaticamente com base nos pinos ligados.
ℹ️ Tags automáticas
Se deixar o campo do nome vazio, o sistema analisa automaticamente os tipos de pino e gera uma tag: ligar "A4/SDA" a "SDA" gera "SDA"; ligar "D3" a "TRIG" gera "DIG_TRIG"; ligar "5V" a "VCC" gera "VCC". Pinos conhecidos (SDA, SCL, TX, RX, MOSI, etc.) são sempre reconhecidos.
⚠️ Fios duplicados
O sistema impede a criação de fios duplicados exatos (mesmo par de pinos de origem e destino). Mas permite ligar múltiplos fios ao mesmo pino — essencial para VCC que alimenta vários componentes, ou SDA/SCL partilhados em I2C.

6 Propriedades do Cabo

Cada fio criado no esquema pode ter propriedades elétricas e físicas associadas — secção do cabo (AWG), cor real, comprimento, tensão e corrente esperadas. Isto transforma o diagrama num documento de engenharia completo.

Ativar o painel

Com a ferramenta ⚡ Ligar pinos ativa, clicar no botão "▸ 📐 Propriedades do cabo" na sidebar. O painel expande com 5 campos editáveis. Pode colapsar novamente quando não for necessário.

💡 Regra prática
Para a maioria dos projetos de prototipagem em eletrónica de baixa potência (sensores, LEDs, MCU), AWG 22 é o standard. Para alimentação de motores e relés, usar AWG 18–16. Para sinais digitais curtos, AWG 26–28 é suficiente.

7 Validação e Avisos

O sistema valida automaticamente cada fio com base nas propriedades elétricas definidas e gera avisos quando deteta potenciais problemas. Todos os avisos são informativos e não-bloqueantes — informam mas permitem continuar.

Tipos de validação

VerificaçãoCondiçãoAviso
Capacidade de correnteCorrente > 100% da capacidade AWG🔴 Vermelho — usar cabo mais grosso
Margem de correnteCorrente entre 80–100% da capacidade🟡 Amarelo — margem reduzida
Queda de tensãoVdrop > 10% da tensão nominal🔴 Vermelho — encurtar cabo ou usar AWG menor
Queda de tensãoVdrop entre 5–10%🟡 Amarelo — aceitável mas monitorar

Como funciona a queda de tensão

A queda de tensão é calculada pela fórmula Vdrop = I × R/m × L × 2 (ida e volta). Por exemplo: um fio AWG 22 (0.053 Ω/m) com 50cm de comprimento a 500mA tem uma queda de 0.5A × 0.053 × 0.5m × 2 = 0.027V (0.5% de 5V — perfeitamente aceitável). Mas o mesmo fio a 5A teria uma queda de 0.27V (5.3% — aviso amarelo).

Painel de avisos

O painel "⚠️ AVISOS" aparece automaticamente na sidebar quando existem alertas. Mostra o número total de avisos e lista cada um com cor de severidade (vermelho, amarelo, azul). Cada aviso indica o cabo afetado e os componentes envolvidos.

⚠️ Avisos na impressão
Ao imprimir o esquema, todos os avisos e recomendações aparecem numa secção dedicada "Avisos e Recomendações" antes das notas. Isto permite que o professor ou o aluno verifiquem os pontos de atenção antes de montar.

8 Recomendações Automáticas

Além da validação elétrica dos cabos, o sistema analisa os componentes do esquema e gera recomendações de boas práticas baseadas no tipo de componente e na forma como está ligado.

Recomendações por componente

Componente / SituaçãoRecomendação
Barramento I2C (SDA/SCL)Verificar resistências pull-up 4.7kΩ no SDA e SCL
Sensores (BME280, DHT22, MPU-6050...)Condensador 100nF entre VCC e GND junto ao sensor
Sensor capacitivo do soloCabo curto (<30cm), par entrançado, evitar fontes de ruído
Relé, motor, solenóideDíodo flyback (1N4007) em paralelo inverso. Alimentação separada.
Servo motorAlimentação separada (não pelo pino 5V). Condensador 100µF.
Motor DC / stepperAlimentação separada obrigatória. GND comum com MCU.
LEDResistência em série (220Ω–1kΩ para 5V, 100Ω–470Ω para 3.3V)
I2C com cabo >100cmLimite prático ~100cm. Considerar I2C extender ou RS-485.
Sinal analógico com cabo >50cmUsar cabo blindado ou par entrançado
SPI com cabo >20cmManter cabos SPI curtos. Reduzir clock se necessário.
MCU (Arduino, ESP, STM32...)Condensador 100nF junto a cada IC. Condensador 100µF na alimentação.
ℹ️ Recomendações vs obrigações
Todas as recomendações são sugestões baseadas em boas práticas de eletrónica. Nem todas se aplicam a todos os cenários — por exemplo, muitos módulos I2C já incluem pull-ups internos. O objetivo é alertar para potenciais problemas, não substituir o julgamento do projetista.
🎓 Valor pedagógico
As recomendações servem como ferramenta de ensino — o aluno aprende por que precisa de um díodo flyback num relé, ou por que o I2C tem limite de distância, no momento exato em que está a projetar o circuito. É aprendizagem contextual.

9 Modos de Visualização

O programa oferece dois modos para visualizar as ligações entre componentes. Pode alternar a qualquer momento sem perder dados.

━ Modo Linhas

As ligações são representadas por curvas Bezier coloridas entre os pinos, com o nome do cabo sobre a linha. É o modo ideal para circuitos simples com poucas ligações — mostra visualmente o percurso de cada fio.

ℹ️ Quando usar Linhas
Circuitos com até ~15–20 ligações. Com mais ligações, as curvas começam a sobrepor-se e o esquema fica difícil de ler.

🏷 Modo Tags

As linhas desaparecem e em cada pino conectado aparece uma etiqueta colorida no formato CABO→Destino (ex: SDA→OLED). Para saber onde um pino liga, basta ler a etiqueta — mostra o nome do cabo e o componente de destino diretamente, sem precisar de procurar.

Este modo é ideal para circuitos complexos — o esquema fica limpo e legível independentemente do número de ligações. É também o modo que melhor se traduz para a tabela de ligações impressa.

🏷 Exemplo de leitura em modo Tags

Se o pino SDA do Arduino tem a tag verde SDA, e o pino SDA do OLED também tem a tag verde SDA, sabemos que estão ligados pelo cabo "SDA" (fio verde).

10 Edição e Navegação

Selecionar e mover

Com a ferramenta ▹ Selecionar ativa, clicar num componente seleciona-o (contorno branco brilhante). Arrastar move-o (o undo só é criado se houver movimento real). O painel "SELECIONADO" na sidebar mostra nome, categoria, pinos, ligações e lista de fios com AWG.

Renomear componente (duplo-clique)

Fazer duplo-clique num componente na ferramenta Selecionar abre um campo de edição inline sobre o header. Permite alterar o nome (ex: mudar "Arduino UNO" para "Arduino UNO #2"). Premir Enter para confirmar ou Escape para cancelar.

Editar pinos de componentes colocados

Com um componente selecionado, o botão "✏️ Editar pinos" na sidebar abre um modal onde se pode adicionar, remover, renomear e mudar o tipo de cada pino — tanto esquerdos como direitos. Isto permite personalizar componentes da biblioteca após colocação, sem necessidade de criar um componente custom.

16 Exemplo Prático Passo a Passo

Vamos criar o esquema de um sistema de monitorização ambiental com Arduino UNO, sensor BME280 (I2C), OLED (I2C) e buzzer de alarme.

Clicar em "+ Adicionar da biblioteca" e selecionar Arduino UNO da categoria Arduino
Repetir para adicionar BME280 I2C (Sensores), OLED 128×64 I2C (Displays) e Buzzer (Atuadores)
Arrastar os componentes para posições lógicas: Arduino à esquerda, sensores ao centro, display à direita
Selecionar a ferramenta ⚡ Ligar pinos
Clicar no preset VCC (vermelho). Ligar o pino 5V do Arduino ao pino VCC do BME280. Repetir para o VCC do OLED e o VCC do Buzzer.
Clicar no preset GND (azul). Ligar GND do Arduino ao GND de cada componente (3 ligações).
Clicar no preset SDA (verde). Ligar A4/SDA do Arduino ao SDA do BME280, depois ao SDA do OLED.
Clicar no preset SCL (amarelo). Ligar A5/SCL do Arduino ao SCL do BME280, depois ao SCL do OLED.
Escrever "BUZZER" no campo de nome, selecionar uma cor laranja. Ligar D8 do Arduino ao pino VCC do Buzzer (que é na verdade o pino de sinal).
Alternar entre Linhas e Tags para ver qual modo fica mais legível com este número de ligações.
Clicar "🖨️ Imprimir esquema" para gerar o documento A4 com a tabela de ligações completa.
ℹ️ I2C: dois componentes no mesmo barramento
O BME280 e o OLED usam ambos I2C — partilham os mesmos fios SDA e SCL. Cada um tem um endereço diferente (BME280: 0x76/0x77, OLED: 0x3C/0x3D). No esquema, é possível ligar ambos ao mesmo pino do Arduino, ou criar dois fios separados SDA→BME e SDA→OLED — ambos são corretos.
